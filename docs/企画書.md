# やりたいことアプリ 企画書

最終更新：2026-07-07

## コンセプト

TwitterやインスタでたSNSを眺めているときに出会った「行きたい美術展」「美味しそうなご飯屋さん」を、その場で気軽にスクラップ。開催日をAIが自動で読み取り、ちょうどいいタイミングでリマインドが届く。**人生でやり残したことが減り、充実感が増えるアプリ。**

肝は2つ：
1. **簡単に入力できる**（見つけた瞬間、3秒で保存）
2. **後からリマインドが来る**（保存して終わり、にならない）

## ターゲットユーザー

- SNSで「行きたい！」と思ったものをスクショやいいねで貯めるが、埋もれて忘れてしまう人
- まずはkanamiさん自身が毎日使いたくなるものを目指す

## MVPの機能範囲

| 機能 | MVP | 備考 |
|---|---|---|
| 共有シートからURL保存 | ✅ | このアプリの肝。Twitter/インスタ/Safariから |
| アプリ内でURL貼り付け・手入力 | ✅ | 共有シートの補完 |
| AIによる日付・タイトル読み取り | ✅ | Claude API。候補表示＋手動で直せるUI |
| リマインド通知 | ✅ | 開始3日前／終了1週間前／日付なしは定期 |
| 「行った！」で完了にする | ✅ | 充実感につながる体験の締め |
| 一覧・詳細画面 | ✅ | |
| カテゴリ分け | ✅ | 展示・ご飯・映画など最小限 |
| 友達と共有・複数端末同期 | ❌ v2以降 | |
| 地図表示 | ❌ v2以降 | |
| スクショ画像からの読み取り | ❌ v2以降 | まずはURLから |

## まだ決めていないこと

- AI読み取りの費用の持ち方（アプリに鍵を埋め込むのは危険なので、小さな中継サーバーを置くか、無料枠の設計をどうするか）→ SwiftUI移植のフェーズで決める
- アプリの正式名称（「やりたいことアプリ」は仮）
- アイコン・配色などのビジュアルアイデンティティ（kanamiさんの領域）
- 完了した「やったこと」の振り返り体験（充実感の演出）をどこまでMVPに入れるか
