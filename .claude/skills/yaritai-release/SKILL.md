# yaritai-release：App Storeリリース手順書

## 前提の説明（kanamiさんに伝えるときの言葉）
- **App Storeに出すには「Apple Developer Program」への登録が必要**：年間99ドル（約1.5万円）の、Appleへの出店料のようなもの。https://developer.apple.com/programs/ から個人で登録できる。審査に1〜2日かかることがあるので、リリース目標日の1週間前までに登録する
- **TestFlight**：正式公開の前に、自分や友達のiPhoneだけにアプリを配れるAppleの仕組み。ベータテスト用。これも開発者登録が必要
- **審査**：Appleが「このアプリを公開してよいか」を人力＋機械でチェックする工程。通常1〜3日

## リリースまでのチェックリスト（この順に案内する）
1. **Apple Developer Program登録**（kanamiさん自身のApple IDで。支払いも本人操作）
2. **App用の素材を準備**（ここはkanamiさんの得意領域）
   - アプリアイコン：1024×1024px、角丸なし・透過なしのPNG
   - スクリーンショット：6.9インチ（iPhone Pro Max）サイズ必須。実機かシミュレータで撮る
   - アプリ名（30文字以内）、サブタイトル、説明文、キーワード
3. **プライバシー関連**
   - 保存したURLをAI読み取りのために外部（Claude API等）へ送る場合、App Privacyで「データ収集」の申告が必要になることがある。提出前に構成を確認して正しく申告する
   - プライバシーポリシーのURLは必須。GitHub Pagesなどで1ページ用意する
4. **通知の許可**：リマインドが肝のアプリなので、通知許可を求めるタイミングと文言（なぜ必要か）を丁寧に設計する。許可を初回起動でいきなり求めると拒否されやすい
5. **Xcodeでの署名設定**：Signing & Capabilities でTeam（開発者アカウント）を選ぶ。Bundle Identifier は `com.kanami.yaritai` のような世界で1つのIDにする。**Share Extension側にも同じ設定が必要**（本体と受付窓口の2つ分）
6. **App Store Connectでアプリ登録**：https://appstoreconnect.apple.com でアプリの入れ物を作り、素材と説明文を入力
7. **アーカイブとアップロード**：Xcodeの Product > Archive → Distribute App。専門用語が多い工程なのでClaudeが画面ごとに案内する
8. **TestFlightで自分のiPhoneで試す**（最低数日、実際にSNSからスクラップして通知が来るまで使ってみる）
9. **審査提出** → リジェクト（差し戻し）されても普通のこと。理由を読んで直せばよい、と安心してもらう

## 審査で気をつけること（このアプリ特有）
- **他サイトの内容を保存するアプリ**なので、サムネイル画像等を表示する場合は権利面に注意。MVPではユーザー自身のメモ＋リンクの形が安全
- **AI（Claude API）を使う機能**：AIが生成・抽出した内容であることが分かるUIにする。API鍵をアプリに埋め込んでいないことを確認（埋め込みはセキュリティ審査以前に危険）
- 最低限の機能しかないアプリは「4.2 Minimum Functionality」で差し戻されることがある → 共有シート保存・AI読み取り・リマインド・完了体験が揃っていれば通る水準。説明文で価値を明確に書く
- 起動時にクラッシュしないこと（審査落ち理由の定番）。提出前に実機で新規インストール→起動を確認

## バージョンアップのとき
- バージョン番号（例 1.0 → 1.1）を上げて、同じ手順でアーカイブ→提出
- 「What's New（このバージョンの新機能）」を書く
